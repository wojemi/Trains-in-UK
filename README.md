# Superstore
This repository contains SQL code describing and analyzing the data concerns 'Trains in UK' using SQL - tickets prices, train journeys, delays and many more. 

# Business problem
The aim of the data analysis is to help identify where delays occur, on which routes train run on time, how ticket prices are structured, and what the level of public demand for train travel is.

# Dataset
Dataset used for this project comes from the KajoData SQL course and contains information about ticket prices, train routes, delays, departure and arrival points.
Dataset includes:
- departure station
- price
- ticket class
- date of journey
- reason for delay
- payment method and many more

# Tools
- SQL - main analysis
- Excel - preliminary data exploration

# Project structure
1. Data - raw dataset
2. SQL - analyzing and drawing conclusions

# Business questions
1. What is the percentage of delayed journeys?
2. What is the average delay time?
3. What is the average ticket price depending on the time of day?
4. Where people travel the most?
5. Which payment method is the most popular?

# Key insights
1. The most popular payment method is by credit card.
2. People travel the most to Liverpool Lime Street.
3. On weekdays there are more delays than in weekends.
4. Tickets are the most expensive in the morning.
5. The longest average delay was 75 minutes.
